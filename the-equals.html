= vs == vs .equals()

Friday, July 29th, 2017 18:36 - PST

Going through school, you've seen '=' a lot. Usually in an algebra equation like, y = (5 + 4) ÷ 3. Luckily for us programmers, it hasn't changed! The '=' is an assignment operator. If we took that algebra equation and put it into our script, we would write int y = (5 + 4) / 3; or because we're smart and we can do that equation in our head, we would write int y = 3;. What the '=' operator tells the compiler is "this variable is equal to this value."

Next we have a funny looking thing, '=='. This is a relational operator. This compares two variables or a variable and a constant. This is normally used in an if/else statement, or while loops. It returns a true or false value. So if we had int y = 3; and int x = 3;. Then, in an if/else statement we wrote if (y == x) the value would return true because y and x are both equal.
