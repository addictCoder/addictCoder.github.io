Into the Void

Friday, July 29th, 2017 18:36 - PST

You see it wit every script you write. That strange and mysterious keyword. Is it trying to send your method into a black hole? What is it's true purpose. Webster's dictionary states that it meant completely empty, but there is so much code that follows it. What does it mean!? Is this some kind of a sick joke? Who would do this to developers? They must be stopped before we all fall victim to their madness!! Why must we write VOID!?

Void does not return a black hole and suck up your code and possibly your whole computer. Instead, void tells the compiler that the method doesn't return any value to it's caller. So there is no return statement. The method simply executes it's code. Here is an example.
